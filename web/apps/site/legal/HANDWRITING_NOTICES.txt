Derived CRNN model
------------------

The exact POC model and metadata are distributed as models/digits-crnn.onnx and
models/digits-crnn.json:

  bea69199be71c01a35f4485ad853ef6fd11608c616c452598cb3f330922db9af  digits-crnn.onnx
  fef02e93f6c412614608ed5c7df24f2b94760057c0c3a857d2e208bfdd34b6fe  digits-crnn.json
  Model bytes: 1714986

The CRNN Tiny architecture was adapted from zjykzj/crnn-ctc:

  Repository: https://github.com/zjykzj/crnn-ctc
  Release: v1.3.0
  Revision: aeceea3a2ab7e973b40d871ff628b327df31a045
  Copyright: 2023 zjykzj
  License: Apache-2.0 (APACHE-2.0.txt)
  Intended initialization checkpoint: crnn_tiny-emnist.pth
  Checkpoint SHA-256: 6d2a653513fd71f9d5de1fc238311dc017d285f0bbc55e09da7ed9eea80479c9

The derivative changes the input width from 160 to 128, composes canonical
one-to-three-digit labels, reports fine-tuning from that intended initialization
reference, and exports a static batch-one ONNX graph. The retained checkpoint
matched the checksum after training, but no checksum was captured at training
start, so pre-training binary lineage is not independently established. See
MODEL_NOTICE.txt and the model metadata for the documented provenance scope and
model limitations.

Training glyphs came from NIST-hosted EMNIST Digits, derived from NIST Special
Database 19. Neither EMNIST nor the upstream checkpoint is distributed here.
